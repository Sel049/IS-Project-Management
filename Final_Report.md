## 3. Lessons Learned

### What went well
- Using GitHub Issues for task tracking improved team accountability.
- Weekly 15-minute standups kept everyone aligned.
- Early prototyping helped catch UI issues before coding.

### What could be improved
- Should have started load testing earlier (Week 5 instead of Week 7).
- Database schema needed 2 revisions – more design time would help.
- Merge conflicts occurred twice – using branches more strictly would avoid this.

### Technical learnings
- Learned to use GitHub Projects for Kanban board.
- Improved Git skills (branching, pull requests, reverting commits).
- Understood importance of `README.md` for documentation.
