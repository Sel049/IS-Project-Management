## 1. Executive Summary
This project aims to develop a web-based course registration system that allows students to browse, add, and drop courses online without visiting the registrar's office. The system will reduce manual errors, save time, and provide real-time seat availability.

## 2. Problem Statement
Currently, the university uses a paper-based or outdated system that causes:
- Long queues during registration week
- Course capacity exceeded due to manual overrides
- Lack of real-time schedule conflicts checking

## 3. Proposed Solution
A responsive web application with:
- Student login & authentication
- Course catalog with available seats
- Add/drop courses with automatic prerequisite check
- Real-time schedule view

## 4. Technology Stack (NEW - Added based on team discussion)
| Layer | Technology | Reason |
|-------|------------|--------|
| Frontend | React.js | Component-based, fast UI updates |
| Backend | Node.js + Express | JavaScript full-stack consistency |
| Database | PostgreSQL | Reliable, supports complex queries |
| Version Control | GitHub | Project management + collaboration |
| Hosting | Render (free tier) | Easy deployment from GitHub |

## 5. Objectives
- Develop a working prototype within 8 weeks
- Reduce registration time by 60%
- Eliminate manual data entry errors
- **NEW:** Support minimum 500 concurrent users during peak registration

## 6. Scope
**In scope:** Student registration, course browsing, schedule generation  
**Out of scope:** Payment integration, faculty grading system

## 7. Key Stakeholders
- Students (primary users)
- Registrar's office
- Academic advisors
